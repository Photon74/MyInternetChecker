# MyInternetChecker

Лёгкое WPF-приложение для периодической проверки доступности интернет-хостов

Описание

Приложение проверяет возможность подключения к списку хостов и предназначено как простой мониторинг наличия интернета. Проект написан на C# (целевой фреймворк .NET 8) и использует WPF для интерфейса.

Особенности

- Проверка доступности списка хостов по интервалу (по умолчанию 1 секунда)
- Конфигурируемый список хостов через файл настроек
- Миграция старого файла настроек (рядом с .exe) в папку `%APPDATA%`
- Есть окно настроек хостов и поддержка автозапуска

Требования

- .NET 8 SDK
- Windows (WPF приложение)

Сборка и запуск

1. Откройте решение в Visual Studio 2022/2023 или используйте CLI
2. Сборка: `dotnet build MyInternetChecker/MyInternetChecker.csproj`
3. Запуск: `dotnet run --project MyInternetChecker/MyInternetChecker.csproj` или запуск из IDE

Настройки

Приложение хранит файл настроек в папке AppData пользователя:

`%APPDATA%\MyInternetChecker\settings.txt`

Каждая непустая строка файла — отдельный хост для проверки. Строки, начинающиеся с `#`, игнорируются. Если файл отсутствует, приложение создаст его с набором хостов по умолчанию:

- `ya.ru`
- `google.com`

Изменение списка хостов

- Правьте `settings.txt` напрямую или используйте окно настроек в приложении (`HostsSettingsWindow`).

Миграция старых настроек

Если в папке с исполняемым файлом присутствует `settings.txt`, а в `%APPDATA%` его нет, приложение автоматически попытается перенести старую конфигурацию в папку `AppData`.

Структура проекта (ключевые файлы)

- `MyInternetChecker/Config.cs` — логика загрузки/миграции настроек
- `MyInternetChecker/PingIt.cs` — проверка доступности хостов
- `MyInternetChecker/HostsSettingsWindow.xaml(.cs)` — UI для редактирования списка хостов
- `MyInternetChecker/AutoStartManager.cs` — управление автозапуском
